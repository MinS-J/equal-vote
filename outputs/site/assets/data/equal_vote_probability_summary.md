# 후보별 득표수 동시 일치 확률 분석 정리

작성일: 2026-06-09

## 1. 문제의 출발점

분석의 출발점은 다음 사건이었다.

```text
인접하거나 유사한 두 선거구에서
후보1 득표수도 같고,
후보2 득표수도 같다.
```

처음에는 특정 사례로 다음과 같은 관내사전투표 결과가 있었다.

```text
송도1동 관내사전투표:
선거인수 4,548
민주당 3,030
국민의힘 1,440
기타 78

송도2동 관내사전투표:
선거인수 4,540
민주당 3,030
국민의힘 1,440
기타 70
```

여기서 핵심은 사람들이 놀라는 이유가 반드시 `3,030표`와 `1,440표`라는 특정 숫자 때문이 아니라는 점이다. 만약 `3,032표 / 1,443표`처럼 다른 숫자였어도, 두 후보의 득표수가 각각 같다는 사실 자체가 놀라움의 대상이 되었을 가능성이 높다.

따라서 사건을 다음 둘로 구분해야 한다.

```text
좁은 사건:
하필 민주당 3,030표, 국민의힘 1,440표가 두 곳에서 동시에 나옴

넓은 사건:
어떤 숫자든 민주당 득표수가 같고, 국민의힘 득표수도 같음
```

뒤의 사건이 실제 의심 또는 놀라움에 더 가깝다.

## 2. 조건부 확률의 중요성

또 하나 중요한 점은 두 선거구의 관내사전투표자 수가 매우 비슷했다는 것이다.

```text
송도1동: 4,548
송도2동: 4,540
차이: 8명
```

민주당과 국민의힘 득표수가 둘 다 같으려면, 두 선거구의 총 사전투표자 수 차이는 전부 기타 쪽에서 흡수되어야 한다.

```text
기타1 - 기타2 = 사전투표자수1 - 사전투표자수2
```

위 사례에서는 기타 차이가 8표면 충분하다.

따라서 사전투표자 수가 이미 관측된 사실이라고 조건화하면, 문제는 다음처럼 바뀐다.

```text
4,548명과 4,540명을 민주당/국민의힘/기타로 나눴을 때,
민주당 표수와 국민의힘 표수가 둘 다 같을 확률은?
```

2026년 시 전체 관내사전투표에서 송도1동/송도2동을 제외한 후보별 분포를 사용하면:

```text
민주당: 67.91%
국민의힘: 30.09%
기타: 2.00%
```

이 조건부 확률은 약 다음과 같았다.

```text
P(민주당 득표수 같음 and 국민의힘 득표수 같음 | 사전투표자수 4548/4540)
≈ 1 / 4,336
```

반면, `3,030표 / 1,440표`라는 특정 숫자까지 고정하면 훨씬 작은 확률이 된다. 하지만 이는 실제 사람들이 놀라는 사건보다 지나치게 좁은 사건 정의일 수 있다.

## 3. 사전투표자 수 유사성까지 포함한 계산

사전투표자 수 자체도 확률변수로 보면 확률은 달라진다.

송도1동/송도2동의 과거 사전투표 성향을 반영했을 때:

```text
P(|송도1동 사전투표자수 - 송도2동 사전투표자수| <= 10)
≈ 9.46%
≈ 1 / 10.6
```

이를 후보별 득표수 동시 일치 확률과 거칠게 곱하면:

```text
0.0946 * (1 / 4,336)
≈ 1 / 45,800
```

즉 실제 관측에 가까운 `±10표 이내` 조건까지 포함해도, 수억분의 일이나 수경분의 일 수준은 아니었다.

또한 사전투표자 수 차이를 `±300표`까지 넓게 허용하고 합산하면, 정규근사 기반으로 다음 정도가 나왔다.

```text
P(|사전투표자수 차이| <= 300 and 민주당/국민의힘 득표수 동시 일치)
≈ 1 / 23,500
```

이는 `±300표`가 넓은 조건이라기보다, 민주당/국민의힘 득표수가 이미 같다면 사전투표자 수 차이는 곧 기타표 차이가 되므로 `±300표` 범위가 사실상 대부분을 포함하기 때문이다.

## 5. 사후탐색 효과

특정 pair 하나를 사전에 정해놓고 보는 확률과, 전국의 많은 pair를 사후적으로 훑어서 하나라도 찾는 확률은 전혀 다르다.

```text
사전 지정 pair:
송도1동/송도2동 하나에서 일어날 확률

사후탐색:
전국의 여러 유사 pair 중 하나라도 발견될 확률
```

가능한 pair가 많으면 생일문제처럼 하나 이상의 일치가 나타날 확률이 커진다.

## 6. Pair 정의

분석에서는 여러 pair 정의를 구분했다.

```text
전국 모든 pair:
가장 넓음. 사후탐색 효과가 매우 큼.

같은 구시군 내 pair:
행정권역을 공유하는 pair.

같은 광역시/도 내 pair:
같은 시도 안의 관내사전투표 row pair.
비교 기회가 크게 늘어나므로 사후탐색 효과가 강해진다.

같은 구시군 + 같은 읍면동 stem pair:
송도1동/송도2동 같은 분할동에 가까운 proxy.
예: OO1동 / OO2동, OO제1동 / OO제2동.
```

마지막 기준은 다음 방식으로 만들었다.

```text
읍면동명에서 숫자 표현을 제거한 stem이 같고,
같은 시도 및 같은 구시군에 속한 row끼리 pair로 묶음.
```

예:

```text
광안제1동 / 광안제4동
신평1동 / 신평2동
송도1동 / 송도2동
```

이 기준에서 2022년 시·도지사 관내사전투표 데이터의 비교 대상은:

```text
1,162 pair
```

## 7. M1: 구시군 공통분포 모델

첫 번째 시뮬레이션 모델 M1은 다음 가정을 두었다.

```text
같은 구시군 안의 모든 row는 같은 관내사전투표율을 가진다.
같은 구시군 안의 모든 row는 같은 후보1/후보2/기타 관내사전 득표율을 가진다.
```

구체적으로:

```text
n_i ~ Binomial(전체선거인수_i, q_구시군)

(후보1, 후보2, 기타)_i
~ Multinomial(n_i, [p1_구시군, p2_구시군, p기타_구시군])
```

M1에서 `같은 구시군 + 같은 읍면동 stem pair` 1,162쌍을 사후적으로 훑은 결과:

```text
사전투표자 수 고정:
하나 이상 일치쌍 발생 확률 ≈ 0.86%

사전투표자 수까지 확률화:
1,000회: 0.7%
3,000회: 1.1%
```

즉 M1에서는 대략:

```text
약 1%
약 1 / 100
```

수준으로 보였다.

## 8. M2: Row 중심 Beta/Dirichlet 모델

M1은 같은 구시군 내 읍면동 차이를 너무 지운다는 한계가 있다. 그래서 M2에서는 각 row의 고유 성향을 중심으로 두고 불확실성을 얹었다.

```text
q_i* ~ Beta(q_i * κ, (1 - q_i) * κ)

p_i* ~ Dirichlet(p_i1 * τ, p_i2 * τ, p_i기타 * τ)

n_i ~ Binomial(전체선거인수_i, q_i*)

득표수_i ~ Multinomial(n_i, p_i*)
```

여기서:

```text
q_i = 해당 row의 실제 관내사전투표율
p_i = 해당 row의 실제 후보1/후보2/기타 관내사전 득표율
```

이 모델은 posterior predictive 성격이 강하다. 즉 선거 전 사전확률이라기보다, 실제 데이터의 지역별 구조를 보존했을 때 이런 패턴이 얼마나 자연스럽게 생기는지를 보는 모델이다.

결과:

```text
κ = τ = 100:
약 0.18%

κ = τ = 500:
약 0.18%

κ = τ = 2000:
약 0.32%
```

즉 M2에서는:

```text
약 0.2% ~ 0.3%
약 1 / 300 ~ 1 / 550
```

정도로 나왔다.

## 9. M2의 문제와 M2b 개선

M2에서 `κ`, `τ`를 모든 row에 동일하게 두면 문제가 있다.

표본 수가 큰 row와 작은 row를 같은 신뢰도로 취급하기 때문이다.

표본 수가 크면 관측 비율이 더 안정적이고, 표본 수가 작으면 관측 비율이 더 튈 수 있다.

따라서 M2b에서는 다음처럼 shrinkage weight `w`를 사용했다.

```text
posterior mean(q_i)
= (1 - w) * row 사전투표율
  + w * 구시군 사전투표율

posterior mean(p_i)
= (1 - w) * row 후보별 득표율
  + w * 구시군 후보별 득표율
```

동시에 표본 수가 큰 row는 posterior가 좁고, 작은 row는 posterior가 넓게 흔들리도록 했다.

즉 `w`는 다음처럼 해석된다.

```text
w = 0.1:
row 고유 성향 90%, 구시군 평균 10%

w = 0.7:
row 고유 성향 30%, 구시군 평균 70%

w = 0.9:
row 고유 성향 10%, 구시군 평균 90%
```

## 10-1. 같은 구시군 전체 pair 기준 추가 결과

위의 `같은 읍면동 stem` 기준은 송도1동/송도2동처럼 이름상 분할동으로 보이는 pair만 남긴 좁은 기준이다. 비교 범위를 조금 넓혀서, 같은 구시군 안의 모든 관내사전투표 row pair를 대상으로도 5,000회 시뮬레이션을 다시 수행했다.

```text
총 26,649쌍
실제 관측 일치 pair: 0쌍
```

```text
같은 구시군 전체 pair까지 사후적으로 훑으면,
후보1/후보2 득표수가 동시에 같은 pair가 하나 이상 나올 확률은
모형에 따라 약 23%~30% 수준까지 올라간다.
```

즉 "같은 구시군 안 어딘가에서 이런 일이 발견되는가"와 "송도1동/송도2동처럼 유사 분할동 pair에서 발견되는가"는 서로 다른 질문이다. 전자는 비교 기회가 훨씬 많기 때문에 확률이 크게 올라간다.

## 10-2. 같은 광역시/도 전체 pair 기준 추가 결과

비교 범위를 더 넓혀서, 같은 광역시/도 안의 모든 관내사전투표 row pair를 대상으로도 5,000회 시뮬레이션을 수행했다.

조건:

```text
2022 시·도지사 관내사전투표
같은 광역시/도 안의 모든 row pair
총 524,784쌍
사전투표자 수까지 확률화
```

## 12. 아직 남은 주의점

현재 결과는 5,000회 수준의 예비 시뮬레이션이다.

특히 M2/M2b는 희귀 사건 영역이라 성공 횟수가 적다.

예:

```text
w = 0.7:
5,000회 중 51회 성공

w = 0.9:
5,000회 중 52회 성공
```

따라서 다음 단계에서는 50,000회 이상으로 반복 수를 늘려 표본오차를 줄이는 것이 좋다.

또한 실제 지리적 인접성이 아니라 읍면동명 stem을 기준으로 pair를 정의했다. 이는 `송도1동/송도2동` 같은 분할동을 잡기 위한 proxy이며, 실제 경계 또는 좌표 기반 인접 pair 목록이 있으면 더 좋다.

## 15. 2022년 사전투표율만 prior로 사용한 2026년 추가 분석

마지막으로, 2026년 분석에서 관내사전투표자 수를 생성할 때 2026년의 실제 사전투표율을 중심으로 두지 않고, 2022년 같은 읍면동의 관내사전투표율만 prior로 사용하는 모델을 추가했다.

이 접근의 동기는 다음과 같다.

```text
사전투표 참여 성향:
지역별로 비교적 유지될 가능성이 있음.

후보별 득표율:
정당이 같더라도 후보, 선거 구도, 이슈가 달라지면 크게 튈 가능성이 있음.
```

따라서 이 추가 모델에서는:

```text
사전투표율 prior:
2022년 같은 시도+구시군+읍면동의 관내사전투표율 사용

후보별 득표율 prior:
2022년 후보별 득표율은 사용하지 않음
2026년 민주당/국민의힘/기타 관내사전 득표율에서 추정
```

구체적으로는 2022년 같은 읍면동이 매칭되는 row에 대해:

```text
q_i ~ Beta(2022 관내사전투표자수_i + 0.5,
           2022 비사전투표자수_i + 0.5)

n_i ~ Binomial(2026 전체선거인수_i, q_i)
```

후보별 득표율은 기존 M2b와 같이 2026년 row별 득표율과 구시군 평균을 shrinkage로 섞었다.

매칭 현황:

```text
2026 관내사전투표 row 수: 3,558
2022 같은 읍면동 사전투표율 매칭 row 수: 2,977
미매칭 row 수: 581
```

## 16. 2026년 두 사건 A·B의 결합확률 (joint probability)

§13의 다음 단계 2번에 해당하는 분석이다. 지금까지는 한 번의 시뮬레이션에서 하나의 pair scope만 셌다. 그러나 다음 두 사건은 서로 독립이 아니다.

```text
A = 같은 읍면동 stem pair에서 동일 득표쌍이 1개 이상 발생
B = 광주+전남 통합 광역권 pair에서 동일 득표쌍이 8개 이상 발생
```

stem pair는 광역권 pair 집합의 부분집합이고(송도1동/송도2동 pair는 인천 광역권 pair에도 포함된다), 같은 구시군의 공통 사전투표 흐름이라는 동일한 잠재요인을 공유한다. 따라서 `P(A) * P(B)`로 단순히 곱하면 안 되고, **같은 시뮬레이션 반복 안에서 두 사건을 동시에 세야** 한다.

이를 위해 `work/simulate_joint_events.py`를 새로 작성했다. 매 반복마다 `sim_a`, `sim_b`를 한 번만 생성한 뒤 두 scope 각각에 대해 동일 득표쌍 수를 세고, 사건 지시변수를 기록한다. 생성 모형은 기존 §14와 동일하다.

조건:

```text
2026 시·도지사 관내사전투표
사건 A: 같은 stem pair (1,214쌍 기반), 일치쌍 >= 1
사건 B: 광주+전남 통합 광역권 pair (577,415쌍 기반), 일치쌍 >= 8
사전투표율 prior: 2026 (current)
반복: 50,000회
실제 관측: A_count = 1 (성립), B_count = 8 (성립)
```

결과:

결과(200,000회로 정밀화):

```text
모델          P(A)     P(B)     P(A∩B)    P(A∩B) 95%CI        P(B|A)  P(A|B)  독립대비
M1 (group)    0.767%   1.485%   0.0295%   [0.0229,0.0380]%    3.85%   1.99%   2.59배
M2b w=0.7     0.587%   0.825%   0.0110%   [0.0073,0.0167]%    1.88%   1.33%   2.27배
M2b w=0.9     0.748%   1.274%   0.0205%   [0.0151,0.0278]%    2.74%   1.61%   2.15배
```

(200,000회 중 A∩B 동시 성공 횟수: M1 59회, w=0.7 22회, w=0.9 41회. 95%CI는 Wilson 구간.)
환산하면 P(A∩B)는 약 1/3,400(M1) ~ 1/9,100(w=0.7) 수준이다.

참고로 50,000회 예비 추정치는 M1 0.038%, w=0.7 0.014%, w=0.9 0.016%였고,
200,000회에서도 같은 자릿수로 안정적이며 신뢰구간이 좁아졌다.

해석:

```text
1. 독립가정 대비 비율 P(A∩B) / (P(A)·P(B))이 모든 모델에서 1보다 크다(약 2.2~2.6배).
   즉 A와 B는 양의 상관이며, 단순 곱으로 계산하면 결합확률을 과소평가한다.

2. 이 상관의 원인은 "iteration마다 공유되는 난수 환경"이 아니다.
   생성 모형에서 각 행(row)은 자기 고정 파라미터(q_alpha/q_beta/dirichlet_alpha)
   로부터 독립적으로 추출된다. 한 iteration 안에 공유되는 잠재 난수는 없다.
   상관의 진짜 원인은 정의상의 포함관계(containment)다.

   사건 A의 scope(같은 stem)는 사건 B의 scope(광역권, 광주+전남 통합)의
   부분집합이다. 따라서 A를 성립시킨 stem 일치쌍은 B의 합계에도 그대로
   포함된다. 즉 countB = countA + countB' 이고, A로 조건을 걸면 B는
   이미 그 일치쌍만큼 채워진 상태에서 시작한다.

3. 진단(30,000회, w=0.7)으로 포함관계를 분리하면 다음과 같다.

   E[B]      = 2.805
   E[B | A]  = 3.954      (A 성립 시 평균 +1.15)
   E[B'|A] - E[B']  ≈ +0.15   (B' = countB - countA, 포함관계 제거)
   corr(countA, B') = 0.006   (사실상 0)

   조건부 평균 상승분 1.15 중 약 1.0(=87%)은 포함관계로 보장되는
   stem 일치쌍 1개 자체이고, stem을 제거한 광역권 합계 B'는 A와
   거의 완전히 독립이다(상관 0.006). 남는 +0.15는 stem 일치가 난
   그 지역(구시군) 안의 비-stem pair가 같은 행 추출값을 공유해
   조금 더 겹치는 약한 국소효과다.

4. 그런데 P(B>=8 | A)는 P(B>=8)의 약 4배까지 뛴다.
   평균은 +1.15밖에 안 움직이는데 꼬리확률이 크게 오르는 이유는,
   B>=8이 평균 2.8 분포의 꼬리에 있어서 포함관계가 보장하는 +1이
   문턱을 "B'>=8"에서 "B'>=7"로 낮추는 효과를 내기 때문이다.
   꼬리에서는 작은 평균 이동이 초과확률을 몇 배로 증폭시킨다.

5. 결합확률 P(A∩B) 자체는 200,000회 기준 약 0.011%~0.030%
   (약 1/3,400 ~ 1/9,100) 수준이다. 모델에 따라 다르며,
   현실적 설정인 M2b w=0.7에서 가장 작다(약 1/9,100, 95%CI 1/6,000~1/14,000).
```

요약하면, A와 B의 양의 상관은 두 사건이 신비롭게 함께 움직여서가 아니라, **B의 정의가 A를 산술적으로 포함**하기 때문이다(+ 꼬리 문턱효과). stem을 제외한 광역권 합계는 A와 거의 독립이다. 그럼에도 실무적 함의는 유지된다. "유사 분할동에서 일치가 발견된 것"과 "광역권 전체에서 일치가 많이 나온 것"을 별개의 독립 증거처럼 곱해 천문학적 확률을 만드는 계산은 부정확하다. 전자는 후자의 부분집합이라 이미 후자에 포함되어 있고, 같은 데이터를 두 번 세는 셈이기 때문이다.
